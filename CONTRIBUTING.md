# Contributing

This fork is focused on making FMI4j usable as a multiplatform JVM runtime for
FMI import.

Contributions are welcome in these areas:

- native runtime builds for missing platforms;
- packaging improvements for Maven and Gradle users;
- CI automation for macOS, Linux, and Windows;
- small reproducible FMU loading examples;
- documentation that helps users verify their runtime architecture.

Please keep changes small and focused. The first maintenance goal is runtime
reliability, not a broad rewrite of FMI4j internals.
