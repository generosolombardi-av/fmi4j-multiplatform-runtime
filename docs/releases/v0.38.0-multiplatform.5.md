# v0.38.0-multiplatform.5

This release is the first cleaned-up community runtime release intended for
early external use.

It preserves the original FMI4j 0.38.0 API surface while publishing a
multiplatform native runtime package under the fork coordinates:

```xml
<dependency>
  <groupId>io.github.generosolombardi.fmi4j</groupId>
  <artifactId>fmi-import</artifactId>
  <version>0.38.0-multiplatform.5</version>
</dependency>
```

## Native Runtime Coverage

The `fmi-import` artifact includes native JNI runtime libraries for:

- macOS universal `darwin64` runtime, supporting both Apple Silicon ARM64 and
  Intel x86_64
- Linux x86_64
- Linux ARM64 / AArch64
- Windows x86_64
- Windows ARM64

## Packaging Changes

- Published `fmi-md` and `fmi-import` to GitHub Packages using the
  `io.github.generosolombardi.fmi4j` group id.
- Removed the runtime SLF4J binding from the published library artifact.
  Applications can now choose their own logging backend, for example Logback in
  Spring Boot, without receiving a duplicate SLF4J provider warning from FMI4j.
- Kept `slf4j-api` as the library logging API dependency.

## Documentation

- Added reproducible build notes for macOS universal runtime packaging.
- Added Linux ARM64 runtime build documentation.
- Added Windows ARM64 runtime build documentation.
- Added GitHub Packages consumption and publication notes.
- Added a Spring Boot smoke example showing how to consume the forked
  dependency from a JVM application.
