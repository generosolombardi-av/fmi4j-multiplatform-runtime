# v0.38.0-multiplatform.6

This release prepares the community FMI4j runtime for Maven Central
publication.

The artifacts are published under the verified Central namespace owned by
Generoso Lombardi:

```xml
<dependency>
  <groupId>io.github.generosolombardi-av.fmi4j</groupId>
  <artifactId>fmi-import</artifactId>
  <version>0.38.0-multiplatform.6</version>
</dependency>
```

## Native Runtime Coverage

The `fmi-import` artifact keeps the multiplatform native JNI runtime coverage
introduced by the previous release:

- macOS universal `darwin64` runtime, supporting both Apple Silicon ARM64 and
  Intel x86_64
- Linux x86_64
- Linux ARM64 / AArch64
- Windows x86_64
- Windows ARM64

## Packaging Changes

- Switched the fork publication group id to
  `io.github.generosolombardi-av.fmi4j`, matching the verified Sonatype Central
  namespace.
- Added a GitHub Actions workflow for publishing signed artifacts to Maven
  Central through the Sonatype Central Portal OSSRH Staging API.
- Kept GitHub Packages publication support available as a secondary channel.
- Kept `slf4j-api` as the library logging API dependency without adding a
  runtime logging backend.

## Documentation

- Added Maven Central consumption and publication documentation.
- Documented the GitHub repository secrets required for Central Portal and GPG
  signing.
- Updated the Spring Boot smoke example to use the Maven Central coordinates
  and to remove the GitHub Packages repository block.
